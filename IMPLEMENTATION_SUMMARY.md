# 实现摘要

本文档记录当前版本的实现边界与已落地能力，用于开发交接与评审。

## 1. 架构概览

- 后端: Flask，路由在 backend/api
- 认证与权限: backend/utils/auth.py
- 数据层: SQLite（业务数据）+ JSON（群签名密钥材料）
- 密钥管理: backend/utils/key_manager.py
- 前端: frontend 静态页面

## 2. 权限与角色

- 角色: admin, user
- 管理员能力:
  - 创建群组
  - 添加成员并分发密钥
  - 追踪签名者身份
  - 查看审计日志
- 普通用户能力:
  - 登录后查看授权范围内资源
  - 对所属群组文件匿名签名
  - 验证签名、声明签名

## 3. 核心业务流程

1) 管理员创建群组并初始化 grpkey/mgrkey/gml
2) 管理员添加成员并完成 JOIN，保存 memkey
3) 成员对文件执行匿名群签名
4) 业务方验证签名有效性
5) 管理员在必要时追踪签名者身份
6) 审计日志记录关键操作

## 4. 测试体系（已统一）

已迁移到 pytest 结构，避免根目录脚本分散：

- 功能测试: tests/functional
- 性能测试: tests/performance
- 共享夹具: tests/conftest.py

规范特性:
- 可重复执行（动态用户名）
- 显式断言与失败退出码
- 统一超时与请求封装
- 性能测试显式开关

## 5. 当前已知边界

- 性能测试为服务级延迟基线，不是压测工具。
- 非空数据库环境下，管理员相关测试需显式提供管理员凭据。
- 密钥与运行数据写入 data 目录，提交前需确保不将运行态数据纳入版本控制。

## 6. 推荐维护动作

- 代码变更涉及 API、权限、测试命令时，同步更新对应文档。
- 保持 docs/README.md 作为文档入口，避免多处重复描述。
